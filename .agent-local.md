## Project-Specific Patterns

### Pack Naming Convention
- Each file in `packs/` defines a language pack; the filename (without `.md`) is the pack name
- Pack names are referenced in `.agent-local.md` via `**Packs**: pack1, pack2, ...`
- `agent-sync.sh` parses the `**Packs**:` line with `sed` — format must be comma-separated, no quotes

### Core Rule Ordering
- Files in `core/` use numeric prefixes for load ordering: `00-`, `10-`, `20-`
- The prefix is stripped for frontmatter lookup (e.g., `00-communication.md` → `communication.yaml`)

### Cursor Frontmatter Mapping
- Each `.yaml` in `templates/rule_templates/cursor_frontmatter/` provides YAML frontmatter for the corresponding rule
- Lookup name = rule filename with numeric prefix and `.md` stripped
- If no matching `.yaml` exists, `agent-sync` generates a default frontmatter

### Skills Distribution
- Source skills live in `skills/` within this repo
- `agent-sync` deploys them to `$PROJECT_DIR/.cursor/skills/` (project root only, not recursive)
- Managed via `.agent-sync-skills-manifest` for precise cleanup
