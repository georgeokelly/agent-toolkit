# todo_list

Tracks planned changes to this repo's functionality — additions, modifications, and removals.

本目录用于记录对本仓库功能的待办变更信息，包括新增、修改、删除相关事项。

## Scope / 记录范围

- New features (Add) / 功能新增
- Enhancements and optimizations (Update / Improve) / 功能调整与优化
- Removals (Remove) / 功能移除
- Tracking items for rule or workflow changes / 规则或流程改动的跟踪项

## Suggested Fields / 建议字段

Each item should ideally include: / 每条事项建议包含以下信息：

- Background and objective / 背景与目标
- Expected changes / 预期变更内容
- Acceptance criteria / 验收标准
- Priority and status (todo / doing / done) / 优先级与状态
- Related commit or issue (optional) / 关联提交或 issue（可选）

## Maintenance / 维护约定

- Descriptions should be actionable and verifiable — avoid vague wording / 描述应可执行、可验证，避免模糊表述
- Completed items may retain a brief conclusion for future reference / 完成项可保留简要结论，便于后续追溯
- Periodically clean up stale, duplicate, or invalidated entries / 定期清理过期、重复或已失效事项
